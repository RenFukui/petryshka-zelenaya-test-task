# Petryshka Zelenaya - System Design & Product Analysis

Проект, посвященный анализу требований, проектированию API и разработке архитектуры для интернет-магазина.

## Описание

В рамках проекта были проработаны ключевые аспекты разработки e-commerce функционала:

- анализ и улучшение требований (корзина)
- проектирование REST API
- разработка архитектуры PUSH-уведомлений

---

## Анализ требований (Cart System)

Проведен аудит требований к корзине интернет-магазина:

- выявлены логические противоречия и нестыковки
- определены риски для бизнеса и разработки
- предложена согласованная версия требований
- сформированы вопросы к заказчику

Подробнее: [task1-requirements-analysis.md](./task1-requirements-analysis.md)

---

## Проектирование API (Partners Screen)

Спроектирован REST API для отображения партнерских магазинов:

- определен endpoint и структура запроса
- разработан формат ответа (JSON)
- учтены переходы на внешние ресурсы

Подробнее: [task2-api-design.md](./task2-api-design.md)

## Архитектура PUSH-уведомлений

Разработана верхнеуровневая архитектура системы уведомлений:

- event-driven подход
- микросервисная архитектура
- обработка различных типов событий (заказы, маркетинг)

Подробнее: [task3-architecture.md](./task3-architecture.md)

## Что демонстрирует проект

- системное мышление
- работа с неоднозначными требованиями
- проектирование API
- понимание архитектуры распределенных систем
